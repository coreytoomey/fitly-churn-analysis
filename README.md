# fitly-churn-analysis
Fit.Ly Churn Analysis - DataCamp Certification Project

This project analyzes customer churn for Fit.Ly, a fictional fitness app used in the DataCamp Data Analyst Certification exam. The goal was to identify the root cause of rising churn and provide actionable recommendations to improve customer retention.

🔍 Project Overview
- Worked with three datasets covering user activity, support interactions, and account status

- Performed data cleaning, validation, and exploratory analysis using Python

- Created visualizations to uncover churn patterns and engagement trends

- Identified the primary drivers of churn and quantified their impact

- Delivered findings through a technical written report and an executive presentation

🛠️ Tools & Skills
- Python (pandas, numpy, matplotlib, seaborn)

- Exploratory Data Analysis (EDA)

- Data cleaning & validation

- KPI definition

- Churn analysis

- Data storytelling & executive communication

📄 Included in This Repository
- workbook.pdf — Full analysis and visualizations

- fitly_slides.pdf — Executive presentation

- fitly.ipynb - Jupyter notebook containing the full Python code

📈 Key Insight (Spoiler)

Churn was primarily driven by slow customer support resolution times, especially during predictable high‑volume windows. Improving support efficiency could significantly reduce churn among passive and at‑risk users.
